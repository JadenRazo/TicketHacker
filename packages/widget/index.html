TicketHacker Widget Test Page

The chat widget should appear in the bottom-right corner of the screen.

Widget Features:

  • Click the chat bubble to open/close the widget
  • Fill out the pre-chat form to start a conversation
  • Send messages and receive real-time responses
  • See typing indicators when agents are responding
  • Rate your experience when the conversation is resolved
  • Conversation persists across page reloads using localStorage

Testing Instructions:

  • Make sure the backend API is running on http://localhost:3001
  • Replace TENANT_ID_HERE below with an actual tenant ID from your database
  • The widget will automatically connect to Socket.IO for real-time updates
  • Open the browser console to see any errors or connection status
Note: Update the data-tenant-id attribute in the script tag below with a valid tenant ID from your database before testing.

API Endpoints Being Used:

  • POST /api/widget/init - Initialize widget configuration
  • POST /api/widget/conversations - Create new conversation
  • GET /api/widget/conversations/:id/messages - Fetch message history
  • POST /api/widget/conversations/:id/messages - Send message
  • POST /api/widget/conversations/:id/typing - Send typing status
  • POST /api/widget/conversations/:id/rate - Submit rating
